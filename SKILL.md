# 日志文件整理技能

这个技能帮助你整理、清理或归档包含日期信息的日志文件。

## 支持的文件命名格式

默认支持的文件名格式：
- `环境名称(YYYY-MM-DD HH_MM_SS).txt`
- 例如：`app(2025-12-17 10_42_06).txt`、`production(2025-01-15 08_30_00).txt`

如果用户的日志使用不同格式，请先确认日期格式，然后调整解析逻辑。

## 工作流程

### 1. 确认目标文件夹

首先确认用户要处理哪个文件夹。如果用户没有指定，询问他们。

### 2. 扫描并解析日志文件

使用 `scripts/scan_logs.py` 脚本扫描文件夹，解析文件名中的日期信息。

```bash
python scripts/scan_logs.py <folder_path>
```

脚本会返回：
- 文件总数
- 日期范围（最早和最晚的文件）
- 文件列表及其解析出的日期

### 3. 确认时间阈值

询问用户要如何筛选文件。理解用户的表达方式：

| 用户说 | 意思 |
|--------|------|
| "删除30天以前的日志" | 保留最近30天，删除更早的 |
| "清理2025年1月之前的文件" | 删除/归档 2025-01-01 之前的文件 |
| "保留最近一周的日志" | 保留最近7天，处理其余的 |
| "移动上个月的日志" | 移动上个月的所有文件 |

**重要**：重复确认时间阈值，避免误操作。例如："确认：将删除 2025-12-17 之前的所有日志文件，共 X 个。继续吗？"

### 4. 确认操作类型

询问用户要对筛选出的文件执行什么操作：

| 操作 | 说明 |
|------|------|
| **归档** | 将文件移动到归档文件夹，原始位置删除 |
| **删除** | 永久删除文件 |
| **仅列出** | 只显示将被处理的文件列表，不执行操作 |

### 5. 执行操作

执行操作前再次确认：

```
即将执行以下操作：
- 操作类型：归档/删除
- 目标文件夹：<路径>
- 时间阈值：<日期>
- 涉及文件：X 个
- 归档位置：<归档路径（如果是归档操作）>

确认执行？
```

对于归档操作，使用 `scripts/archive_logs.py`：
```bash
python scripts/archive_logs.py <source_folder> <archive_folder> --before-date <YYYY-MM-DD>
```

对于删除操作，使用 `scripts/delete_logs.py`：
```bash
python scripts/delete_logs.py <folder_path> --before-date <YYYY-MM-DD>
```

### 6. 报告结果

执行完成后，向用户报告：
- 处理的文件数量
- 失败的文件（如果有）
- 释放的磁盘空间（如果可以计算）

## 注意事项

- **安全第一**：删除操作不可逆，务必多次确认
- **文件名解析失败**：如果某些文件名无法解析日期，列出这些文件让用户确认
- **重复处理**：如果归档文件夹已存在同名文件，询问用户如何处理（覆盖/跳过/重命名）
- **空文件夹**：处理完成后，询问是否删除原始空文件夹
